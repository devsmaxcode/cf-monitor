File: `src/routes/_dashboard.tsx:7`, `src/components/dashboard/dashboard-context.tsx:95`, `src/components/dashboard/dashboard-consumers.ts:73`, `src/lib/monitor.server.ts:876`

Issue: The `_dashboard` layout loader calls `getDashboardFn`, which calls `buildMetrics` and reads all metric rows. On mount, `DashboardProvider` immediately calls `loadDashboard` again. The Metrics consumer then calls `getMetricRowsPageFn` for server-side paging. // src/routes/_dashboard.metrics.tsx

### 3. Metric batch/time column logic is duplicated in server and client helpers

File: `src/lib/monitor.server.ts:1026`, `src/lib/monitor.server.ts:1052`, `src/lib/monitor.server.ts:1073`, `src/components/dashboard/helpers.ts:327`, `src/components/dashboard/helpers.ts:402`, `src/components/dashboard/helpers.ts:387`

Issue: `metricTimeColumns`, `metricTimeColumn`, `metricBatchColumn`, and `batchTimeRange` exist in both server and dashboard helper layers with nearly identical logic. The formatting already differs slightly (`appTimeLabel`/comma on the server versus local helper formatting in the client), which is drift caused by copying.

Severity: High

Fix: Move the shared logic and `MetricTimeColumn` type into a single client-safe module, then import it from both server and dashboard code.

```ts
// src/lib/metric-time.ts
export type MetricTimeColumn = {
  key: string
  label: string
  meta: string
  sort: number
}

export function metricBatchColumn(row: Pick<Record<string, string>, 'timestamp_utc' | 'round' | 'round_id'>): MetricTimeColumn {
  const column = metricTimeColumn(row.timestamp_utc || '')
  const roundValue = row.round || row.round_id
  if (!roundValue) return column

  const roundText = String(roundValue)
  const recheck = roundText.endsWith('-recheck')
  const round = recheck ? roundText.replace(/-recheck$/, '') : roundText
  return { ...column, key: `batch-${roundText}`, label: recheck ? `Recheck ${round}` : `Round ${round}` }
}
```
